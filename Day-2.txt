What are the differences between File Management System(FMS) and Database Management System(DBMS)
--------------------------------------------------------------------------------------------------
1. High data redundancy(duplicate data).
   reduce data redundancy.
2. Difficult to maintain and update.
   Easy to maintain and update.
3. Security is limited.
   Provides strong security mechanism.
4. Data sharing is difficult.
   multiple users can access data simultaneously.
5. No relationship between files.
   Supports relationships between tables.
6. Backup and recovery is difficult.
   provides backup and recovery features.
7. Suitable for small applications.
   Suitable for small, medium and large applications.

Q. What is the difference between file system and a DBMS?(IQ)
--------------------------------------------------------------
A file system stores data in separate files may lead to data redundance, and security issues. A DBMS stores data in a structured manner using tables, reduces redundancy, provides security, supports multiple users, and ensures data consistency and integrity.

Q. What is Data, field, record, database
-------------------------------------------
What is Data?
--------------
raw facts or figures without context 
example: 101,pari,60000,hyd

what is a field?
---------------
smallest unit of data in database(column/attribute)
example: empid, empname, empsalary, empaddress.

what is a record?
----------------
collection of related fields(row/tuple).
example:(101,pari,60000,hyd);

what is database?
------------------
organised collection of related records stored together.
example: in employee database contains all employee details.

what is DBMS
-------------
A DBMS is a software used to create, store, retrieve, update, and manage databases.


Client-Server architecture
---------------------------
In client-server architecture, there are 3 components such as:
Client
Server
Protocol

Client: The Role of the client in Client-Server arch is send request to the server. and to get response from the server.

Server: Server is a software, the role of the sever in client-server-arch is to take request from clients if the request is valid request then it will generate some response, we can handover that response to client. If the client request is not a valid request, then it will raise errors.

Protocol: It has a set of rules and regulations, The role of Protocol in client-server architecture is to send request from client to server.

Data Models in DBMS
--------------------
A data model defines how data is stored, organised and manipulated in a database.

1. Physical data model 
------------------------
def: It describes how data is physically stored in storage devices.
audience: Database Administrators

2. Logical data model
----------------------
def: It defines the structure of data (entities, attributes, relationships)
independent of physical storage areas.

audience: developers and database architects.

3.Hierarchical Data model
--------------------------
def: Data is represented as a tree structure.(parent child relationship).
example: Company-->depts-->employees.

4.Network Data model
---------------------
def: Data is represented as graph with records(nodes) and relationships(edges).
example: Student enrolled in multiple courses, courses taught by multiple professors.

5.Relational Data Models(RDBMS)
--------------------------------
Data is stored in tables(relations) with rows(tuple) and columns(attributes).
example:
---------
student(id,name,marks,dept_id)
departments(dept_id,dept_name)
